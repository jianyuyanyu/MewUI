# Code of Conduct

## 기본 원칙

MewUI는 아직 빠르게 변화하는 실험적 오픈소스 프로젝트입니다.
이 저장소는 사용자, 기여자, 유지보수자 모두에게 **환영받을 수 있고**, **기술적으로 집중되어 있으며**, **지속 가능하게 운영되는 공간**이어야 합니다.

이 프로젝트에 참여하는 모든 사람은 서로를 존중하고, 논의를 건설적으로 유지하며, 프로젝트의 방향성과 운영 한계를 함께 존중해야 합니다.

## 기대하는 행동

다음과 같은 행동은 프로젝트에 도움이 됩니다.

- 이슈, PR, 리뷰, 토론에서 예의 있고 차분하게 소통합니다.
- 문제 제기 시 재현 방법, 환경, 기대 동작, 실제 동작 등 구체적인 맥락을 제공합니다.
- 의견 충돌이 있을 때 상대의 악의를 가정하지 않고 먼저 의도와 배경을 확인합니다.
- 피드백은 개인이 아니라 코드, 설계, 동작, 범위에 초점을 맞춥니다.
- 유지보수자의 시간, 우선순위, 유지보수 비용, 장기적인 설계 방향을 존중합니다.
- 자신에게 필요한 요구사항이라도 프로젝트 전체의 방향과 적합성을 함께 고려합니다.
- 특정 변경이 거절되거나 보류되더라도 이를 개인적 공격으로 받아들이지 않습니다.
- 같은 주제를 다시 제기할 때는 기존 결정, 이전 논의, 새 근거를 함께 정리합니다.
- 커스터마이징, 외부 확장, 별도 라이브러리, 애플리케이션 레벨 해결책이 가능한 경우 그 가능성도 함께 검토합니다.

## 허용되지 않는 행동

다음은 대표적인 예시이며, 핵심은 사람과 프로젝트 운영을 해치는 방식의 참여를 허용하지 않는다는 점입니다.

- 괴롭힘, 위협, 차별, 모욕, 조롱, 인신공격 등 사람에 대한 공격적 행동
- 적대적 표현, 공개적 비난, 감정적 압박, 반복적 도발 등으로 논의를 과열시키는 행동
- 이미 제시된 답변, 결정, 대안을 무시한 채 동일 요구를 반복하거나 여러 채널에서 밀어붙이는 행동
- 개인적 선호나 특정 애플리케이션 요구를 프로젝트의 기본 방향으로 지속적으로 강요하는 행동
- 동의 없는 개인정보 또는 비공개 정보 공개, 기타 프로젝트 운영을 과도하게 어렵게 만드는 행동

## 프로젝트 방향과 범위 존중

MewUI는 모든 요청을 수용하는 범용 요청 처리 창구가 아닙니다.

- 모든 제안이 프로젝트의 현재 로드맵과 맞는 것은 아닙니다.
- 어떤 아이디어는 유용하더라도 코어에 포함되지 않을 수 있습니다.
- 어떤 요구사항은 사용자 코드, 샘플, 확장 라이브러리, 또는 별도 구현으로 남는 것이 더 적절할 수 있습니다.
- 유지보수자가 사전 논의를 요청하거나 제안을 거절하는 것은 프로젝트 운영과 설계 판단의 일부입니다.

제안이 받아들여지지 않았다는 사실 자체를 부당함이나 적대의 근거로 해석해서는 안 됩니다.

좋은 참여는 원하는 결과를 얻는 것만이 아니라, 결과와 무관하게 프로젝트의 판단 구조를 존중하는 태도까지 포함합니다.

## 소통 채널 사용 원칙

각 채널은 목적에 맞게 사용되어야 합니다.

- **Issue**: 재현 가능한 버그, 명확한 문제 보고, 방향 확인이 필요한 제안
- **Pull Request**: 합의된 범위의 변경, 또는 영향이 제한적인 수정
- **Discussion**: 방향성 검토, 설계 대화, 탐색적 아이디어

유지보수자는 필요하다고 판단할 경우 논의를 더 적절한 채널로 이동하도록 요청할 수 있습니다.

## 유지보수자 책임

유지보수자는 프로젝트의 건강성과 지속 가능성을 위해 커뮤니티 공간을 관리할 책임이 있습니다.

필요한 경우 다음과 같은 조치를 취할 수 있습니다.

- 논의를 다른 채널로 이동하도록 요청
- 범위를 벗어나거나 시기상조인 이슈/PR 종료
- 설계 방향과 맞지 않는 제안에 대해 보류 또는 거절
- Code of Conduct 위반 콘텐츠 편집, 숨김, 삭제
- 반복적 또는 중대한 위반에 대해 경고, 잠금, 참여 제한 또는 차단

## 적용 범위

이 Code of Conduct는 다음을 포함한 프로젝트 관련 공간 전반에 적용됩니다.

- Issues
- Pull requests
- Discussions
- Code review comments
- 저장소와 연결된 기타 커뮤니티 상호작용

## 신고

이 Code of Conduct 위반을 경험하거나 확인한 경우, 가능하면 공개 충돌을 키우기보다 저장소에 명시된 연락 수단이 있다면 이를 통해 저장소 소유자 또는 유지보수자에게 비공개로 알려주세요.

저장소에 별도의 비공개 연락 수단이 명시되어 있지 않다면, 불필요한 공개 충돌을 피할 수 있는 범위에서 가장 적절한 채널을 선택해 상황을 알려 주세요.

신고 시에는 검토에 도움이 되도록 관련 링크, 스크린샷, 인용, 시간 정보 등 맥락을 함께 제공해 주세요.

## 집행

유지보수자는 신고 및 상황을 선의로 검토하며, 프로젝트와 커뮤니티 보호에 필요하다고 판단하는 방식으로 대응합니다.

가능한 대응에는 다음이 포함될 수 있습니다.

- 비공식 안내 또는 공식 경고
- 댓글/콘텐츠 수정 또는 제거
- 스레드 잠금
- 기여 거절
- 일시적 또는 영구적 참여 제한

## 마지막 원칙

오픈소스 참여는 권리이면서도 관계입니다.
서로에 대한 존중뿐 아니라, 프로젝트의 한계와 운영 부담을 존중하는 태도 역시 이 저장소의 중요한 규범입니다.

이 저장소는 금지 목록을 최소화하는 대신, 좋은 참여가 무엇인지 더 분명히 공유하는 방향을 지향합니다.
